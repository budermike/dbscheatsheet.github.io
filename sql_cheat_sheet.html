Grundstruktur
SELECT … FROM … WHERESFW-Block
Grundbaustein jeder SQL-Query.
SELECT stud_no, name FROM student WHERE stud_no < 3;
SELECT * gibt alle Attribute zurück. SQL entfernt Duplikate NICHT automatisch.
DISTINCT
Entfernt doppelte Tupel aus dem Ergebnis.
SELECT DISTINCT date_of_birth FROM student;
Filterung & Prädikate
BETWEEN
Wertebereich (inklusiv).
WHERE date_of_birth BETWEEN '1992-01-01' AND '1994-01-01'
LIKE
Muster-Vergleich. % = beliebig viele Zeichen, _ = genau ein Zeichen.
WHERE name LIKE 'Sm%'
AND / OR / NOT
Logische Verknüpfung von Prädikaten.
WHERE age > 18 AND NOT name = 'admin'
Joins & Mengenoperationen
JOIN (implizit)
Verbindet mehrere Relationen im FROM mit Bedingung im WHERE.
SELECT * FROM student S, attends A WHERE S.stud_no = A.stud_no;
UNION / UNION ALL
Vereinigung. UNION entfernt Duplikate, UNION ALL behält sie.
SELECT * FROM prof1 UNION SELECT * FROM prof2;
INTERSECT
Schnittmenge beider Ergebnisse.
SELECT * FROM prof1 INTERSECT SELECT * FROM prof2;
EXCEPT
Differenz — Tupel die nur in der ersten Relation vorkommen.
SELECT * FROM prof1 EXCEPT SELECT * FROM prof2;
Aggregation & Groupierung
Aggregatfunktionen
COUNT(), SUM(), AVG(), MAX(), MIN() — fassen Werte einer Spalte zusammen.
SELECT count(*) FROM student; SELECT avg(credits) FROM lecture;
GROUP BY
Teilt Tupel in Gruppen auf, aggregiert jede Gruppe separat. Alle nicht-aggregierten Attribute im SELECT müssen im GROUP BY stehen.
SELECT course_no, count(*) AS no_of_students FROM attends GROUP BY course_no;
HAVING
Filtert Gruppen nach der Aggregation (WHERE gilt vor GROUP BY, HAVING danach).
SELECT prof_pers_no, count(*) AS no_of_lect FROM lecture GROUP BY prof_pers_no HAVING count(*) > 3;
Sortierung & Subqueries
ORDER BY
Sortiert Ergebnis (ASC = aufsteigend = Standard, DESC = absteigend).
SELECT * FROM student ORDER BY date_of_birth DESC, name;
IN (unkorre. Subquery)
Prüft ob Wert in Ergebnis einer inneren Query ist. Innere Query läuft einmal.
SELECT name FROM student S WHERE S.stud_no IN ( SELECT stud_no FROM attends WHERE course_no = 5 );
EXISTS (korre. Subquery)
True wenn innere Query mindestens ein Tupel liefert. Innere Query läuft pro Outer-Tupel.
WHERE EXISTS ( SELECT * FROM assistant B WHERE B.mgr = P.pers_no AND NOT B.area = A.area );
Views & temporäre Views
CREATE VIEW
Erstellt eine virtuelle Relation. Update nur möglich wenn: 1 Base-Relation, Key vorhanden, kein Aggregat/GROUP BY/DISTINCT.
CREATE VIEW aboveAvgCredits AS SELECT course_no, prof_pers_no FROM lecture WHERE credits > (SELECT avg(credits) FROM lecture);
WITH (temporäre View)
Existiert nur für die aktuelle Query. Mehrere durch Komma trennbar.
WITH aboveAvgCredits(cno, pno) AS ( SELECT course_no, prof_pers_no FROM lecture WHERE credits > (SELECT avg(credits) FROM lecture) ) SELECT * FROM aboveAvgCredits;
WITH RECURSIVE
Berechnet transitive Hüllen. Startet mit Initial-Query, führt rekursive Query aus bis keine neuen Zeilen hinzukommen.
WITH RECURSIVE req(courseno) AS ( SELECT prerequisite FROM requires WHERE advanced = 5259 UNION SELECT r.prerequisite FROM requires r, req q WHERE q.courseno = r.advanced ) SELECT * FROM req;
Daten einfügen
INSERT INTO … VALUESvollständig
Fügt Tupel mit allen Attributen ein.
INSERT INTO professor VALUES(123456, 'Kossmann', 012);
INSERT INTO … VALUESpartiell
Nicht angegebene Attribute erhalten NULL (oder Default-Wert). Key muss angegeben werden.
INSERT INTO professor(pers_no, name) VALUES(123456, 'Kossmann');
INSERT INTO … SELECTaus Relation
Kopiert Daten aus einer anderen Relation.
INSERT INTO professor(pers_no, name) SELECT pers_no, name FROM assistant WHERE pers_no = 111111;
Daten löschen
DELETE FROM … WHERE
Löscht alle Tupel die das Prädikat erfüllen.
DELETE FROM professor WHERE pers_no = 123456;
Ohne WHERE werden ALLE Tupel gelöscht!
Daten ändern
UPDATE … SET … WHERE
Ändert Attributwerte der Tupel die WHERE erfüllen. SET bestimmt den neuen Wert.
UPDATE professor SET office_no = 121 WHERE pers_no = 123456;
Relationen erstellen
CREATE TABLE
Erstellt eine neue Relation mit Schema.
CREATE TABLE professor ( pers_no integer, name varchar(80), office_no integer );
Constraints
PRIMARY KEY
Definiert den Primärschlüssel (implizit NOT NULL + UNIQUE).
PRIMARY KEY (pers_no)
NOT NULL
Attribut muss immer einen Wert haben.
name varchar(80) NOT NULL
UNIQUE
Keine doppelten Werte erlaubt (kann NULL sein, im Gegensatz zu PRIMARY KEY).
office_no integer UNIQUE
CHECK
Beliebiges Prädikat als Constraint — kann auch Subquery enthalten.
CHECK (office_no > 0 AND office_no < 99999)
DEFAULT
Gibt einen Standardwert wenn kein Wert angegeben wird.
area varchar(200) DEFAULT 'computer science'
Referentielle Integrität
FOREIGN KEY … REFERENCES
Jeder Fremdschlüsselwert muss als Primärschlüssel in der referenzierten Relation existieren.
FOREIGN KEY (prof_pers_no) REFERENCES professor(pers_no)
ON DELETE / ON UPDATE
Verhalten bei Änderung des referenzierten Schlüssels.
ON DELETE SET NULL -- FK wird NULL ON DELETE CASCADE -- abhängige Tupel werden mitgelöscht -- (kein RESTRICT = Fehler)
SET NULL setzt FK auf NULL. CASCADE propagiert Löschung/Änderung.
Index & Objekte entfernen
CREATE INDEX
Erstellt einen Index auf einem Attribut für schnellere Abfragen (verlangsamt Updates).
CREATE [UNIQUE] INDEX idx_name ON relation (attribut [ASC|DESC]);
DROP
Entfernt Relationen, Views oder Indexe vollständig.
DROP TABLE relation_name; DROP VIEW view_name; DROP INDEX index_name;
