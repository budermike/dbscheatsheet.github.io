Database Systems — Chapter 4
DRL · DML · DDL
Grundstruktur
SELECT … FROM … WHERESFW-Block
Grundbaustein jeder SQL-Query. SQL entfernt Duplikate nicht automatisch (Bag-Semantik).
SELECT stud_no, name FROM student WHERE stud_no < 3;
DISTINCT
Entfernt doppelte Tupel aus dem Ergebnis.
SELECT DISTINCT date_of_birth FROM student;
Filterung & Prädikate
BETWEEN
Wertebereich (inklusiv an beiden Enden).
WHERE date_of_birth BETWEEN '1992-01-01' AND '1994-01-01'
LIKE
Muster-Vergleich. % = beliebig viele Zeichen, _ = genau ein Zeichen.
WHERE name LIKE 'Sm%' -- startet mit "Sm" WHERE name LIKE '_ones' -- ein beliebiges Zeichen, dann "ones"
AND / OR / NOT
Logische Verknüpfung von Prädikaten im WHERE.
WHERE age > 18 AND NOT name = 'admin'
Joins & Mengenoperationen
JOIN (implizit)
Verbindet Relationen im FROM, Join-Bedingung im WHERE. Aliases vermeiden Namenskonflikte.
SELECT * FROM student S, attends A, lecture L WHERE S.stud_no = A.stud_no AND A.course_no = L.course_no;
UNION / UNION ALL
Vereinigung. UNION entfernt Duplikate automatisch, UNION ALL behält sie.
SELECT * FROM prof1 UNION SELECT * FROM prof2;
INTERSECT
Schnittmenge — nur Tupel die in beiden Ergebnissen vorkommen.
SELECT * FROM prof1 INTERSECT SELECT * FROM prof2;
EXCEPT
Differenz — Tupel die nur in der ersten, nicht in der zweiten Relation vorkommen.
SELECT * FROM prof1 EXCEPT SELECT * FROM prof2;
Aggregation & Gruppierung
Aggregatfunktionen
Fassen Spaltenwerte zu einem einzelnen Wert zusammen.
SELECT count(*) -- Anzahl Tupel SELECT count(DISTINCT x) -- Anzahl eindeutiger Werte SELECT sum(credits) -- Summe SELECT avg(credits) -- Durchschnitt SELECT max(stud_no) -- Maximum SELECT min(stud_no) -- Minimum
GROUP BY
Teilt Tupel in Gruppen, aggregiert jede separat. Alle nicht-aggregierten Attribute im SELECT müssen im GROUP BY stehen.
SELECT course_no, count(*) AS no_of_students FROM attends GROUP BY course_no;
HAVING
Filtert Gruppen nach der Aggregation. WHERE läuft vor GROUP BY, HAVING danach.
SELECT prof_pers_no, count(*) AS no_of_lect FROM lecture GROUP BY prof_pers_no HAVING count(*) > 3;
Sortierung & Subqueries
ORDER BY
Sortiert das Ergebnis. Standard ist ASC (aufsteigend). Mehrere Attribute möglich.
SELECT * FROM student ORDER BY date_of_birth DESC, name ASC;
IN — unkorrelierte Subquery
Innere Query läuft einmal, unabhängig von der äusseren. Prüft ob Wert im Ergebnis enthalten ist.
SELECT name FROM student S WHERE S.stud_no IN ( SELECT stud_no FROM attends WHERE course_no = 5 );
EXISTS — korrelierte Subquery
Innere Query läuft für jedes Tupel der äusseren Query neu. EXISTS ist true wenn Subquery ≥ 1 Tupel liefert.
SELECT DISTINCT P.name FROM professor P, assistant A WHERE A.mgr = P.pers_no AND EXISTS ( SELECT * FROM assistant B WHERE B.mgr = P.pers_no AND NOT B.area = A.area );
Views & temporäre Views
CREATE VIEW
Erstellt eine persistente virtuelle Relation. Update nur möglich wenn: eine Base-Relation, Key vorhanden, kein Aggregat / GROUP BY / DISTINCT.
CREATE VIEW aboveAvgCredits AS SELECT course_no, prof_pers_no FROM lecture WHERE credits > (SELECT avg(credits) FROM lecture);
WITH — temporäre View
Existiert nur für die aktuelle Query. Mehrere temporäre Views durch Komma trennbar.
WITH aboveAvgCredits(cno, pno) AS ( SELECT course_no, prof_pers_no FROM lecture WHERE credits > (SELECT avg(credits) FROM lecture) ) SELECT * FROM aboveAvgCredits;
WITH RECURSIVE
Berechnet transitive Hüllen. Startet mit Initial-Query, führt rekursive Query iterativ aus bis keine neuen Zeilen hinzukommen.
WITH RECURSIVE req(courseno) AS ( SELECT prerequisite FROM requires -- initial WHERE advanced = 5259 UNION SELECT r.prerequisite -- rekursiv FROM requires r, req q WHERE q.courseno = r.advanced ) SELECT * FROM req;
Daten einfügen
INSERT INTO … VALUESvollständig
Fügt ein Tupel mit Werten für alle Attribute ein.
INSERT INTO professor VALUES(123456, 'Kossmann', 012);
INSERT INTO … VALUESpartiell
Nur angegebene Attribute werden gesetzt, der Rest erhält NULL (oder DEFAULT). Key muss immer angegeben werden.
INSERT INTO professor(pers_no, name) VALUES(123456, 'Kossmann'); -- → (123456, 'Kossmann', NULL)
INSERT INTO … SELECTaus Relation
Kopiert Daten aus einer anderen Relation (oder Query-Ergebnis).
INSERT INTO professor(pers_no, name) SELECT pers_no, name FROM assistant WHERE pers_no = 111111;
Daten löschen
DELETE FROM … WHERE
Löscht alle Tupel die das Prädikat erfüllen.
DELETE FROM professor WHERE pers_no = 123456;
Ohne WHERE werden ALLE Tupel der Tabelle gelöscht!
Daten ändern
UPDATE … SET … WHERE
Ändert Attributwerte der Tupel die WHERE erfüllen. Mehrere Attribute im SET durch Komma trennbar.
UPDATE professor SET office_no = 121 WHERE pers_no = 123456;
Relationen erstellen
CREATE TABLE
Erstellt eine neue Relation mit Attributen und Datentypen.
CREATE TABLE professor ( pers_no integer, name varchar(80), office_no integer );
Constraints
PRIMARY KEY
Definiert den Primärschlüssel. Impliziert automatisch NOT NULL und UNIQUE.
CREATE TABLE professor ( pers_no integer NOT NULL PRIMARY KEY, name varchar(80) );
NOT NULL
Attribut muss bei jedem INSERT einen Wert haben.
name varchar(80) NOT NULL
UNIQUE
Keine doppelten Werte erlaubt. Im Gegensatz zu PRIMARY KEY kann UNIQUE-Attribut NULL sein.
office_no integer UNIQUE
CHECK
Beliebiges Prädikat als Constraint — kann auch eine vollständige SQL-Subquery enthalten.
CHECK (office_no > 0 AND office_no < 99999) -- mit Subquery: CHECK (stud_no NOT IN ( SELECT e.stud_no FROM examines e WHERE e.course_no = attends.course_no AND grade >= 18 ))
DEFAULT
Standardwert wenn kein Wert im INSERT angegeben wird. Ohne DEFAULT → NULL.
area varchar(200) DEFAULT 'computer science'
Referentielle Integrität (Foreign Key)
FOREIGN KEY … REFERENCES
Jeder FK-Wert muss als PK in der referenzierten Relation existieren — oder NULL sein.
CREATE TABLE lecture ( course_no integer PRIMARY KEY, prof_pers_no integer NOT NULL, FOREIGN KEY (prof_pers_no) REFERENCES professor(pers_no) );
ON DELETE / ON UPDATE
Verhalten wenn der referenzierte Primärschlüssel geändert oder gelöscht wird.
REFERENCES professor(pers_no) ON DELETE SET NULL -- FK → NULL ON UPDATE CASCADE -- FK-Wert wird mitgeändert -- ON DELETE CASCADE → abhängige Tupel werden mitgelöscht -- (kein Keyword) → RESTRICT: Fehler wenn FK noch referenziert
Index & Objekte entfernen
CREATE INDEX
Erstellt einen Index für schnellere Abfragen. DBMS erstellt automatisch einen Index auf dem PK. Vorsicht: verlangsamt Updates.
CREATE [UNIQUE] INDEX idx_name ON relation (attribut [ASC|DESC]);
DROP
Entfernt Relationen, Views oder Indexe vollständig und unwiderruflich.
DROP TABLE relation_name; DROP VIEW view_name; DROP INDEX index_name;
